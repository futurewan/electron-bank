## Why

当前应用存在多处 UI 冗余和流程不合理的问题：首页按钮样式不够突出、包含无用功能（导入付款人映射）、数据统计模块与实际使用场景不符；对账管理页面展示了不必要的状态和创建功能；文件夹配置流程需要用户分别设置多个文件夹，操作繁琐；生成的报告缺少规范的 Excel 格式和标准化的字段定义。本次改动旨在简化用户操作流程、优化 UI 展示、建立标准化的本地文件夹结构和规范化的报告生成体系。

## What Changes

### 1. 首页 (Home) 改动
- **放大「开始对账」按钮**，使其更加醒目突出
- **移除按钮中的 "+" 图标**
- **移除「导入付款人映射」按钮**
- **移除数据统计模块**（待对账、已完成、异常项、匹配率四个统计卡片）
- **「待完成任务」模块改为「已完成任务」模块**，展示 `status === 'completed'` 的批次
- **移除任务卡片中的状态标签**（因为已完成任务不需要再显示状态）

### 2. 对账管理 (Reconciliation) 改动
- **移除状态标签 (Tag)**，该页面仅展示已匹配完成的历史记录
- **移除「创建批次」按钮**，用户不需要在此页面手动创建批次
- **移除创建批次弹窗 (Modal)**，以及相关逻辑

### 3. 对账详情 (ReconciliationDetail) 改动
- **匹配统计数据模块改为三项**：
  - 完美匹配 XX（合并原来的完美匹配 + 容差匹配 + 代付匹配 + AI 里的匹配上的数据）
  - 其中可解释性 XX（原来的容差 + 代付 + AI 匹配中需要解释的部分）
  - 异常情况 XX（异常条目的数量）
- **异常检测结果以表格形式展示**，字段与异常情况处理报告一致：风险等级 | 异常类型 | 银行流水详情 | AI诊断分析 | AI建议操作
- **展示自动生成的报告名称列表**，每个报告名称旁配有对应的「下载」按钮，供用户直接下载具体的 Excel 报告文件
- **移除「批次信息」卡片**（右侧的批次 ID、创建时间等信息）
- **保留剩余银行流水和剩余发票统计**

### 4. 本地文件夹生成逻辑改动 **BREAKING**
- **改为只选择一个根文件夹**：用户首次点击「开始对账」时，弹窗提示选择一个本地文件夹，下面的这些文件夹都保存在这个选中的本地文件夹内
- **系统自动生成 3 个子文件夹**：
  - `00归档` — 每次对账完成后自动生成以日期+次序命名的子文件夹（如 `20260215-1`），其中包含 3 个子文件夹：`发票`、`银行流水`、`AI比对报告`
  - `01发票` — 用户存放发票文件的位置
  - `02银行流水` — 用户存放银行流水数据的位置
- **对账流程自动化**：点击「开始对账」后，系统自动抓取 `01发票` 和 `02银行流水` 文件夹内的数据进行比对
- **自动归档**：比对完成后，系统自动将 `01发票` 和 `02银行流水` 中的数据转存到 `00归档` 对应的子文件夹中
- **自动生成报告**：AI 比对报告自动存储到 `00归档` 对应子文件夹的 `AI比对报告` 目录中
- **设置页面可重置初次选择的本地文件夹的位置**

### 5. 报告生成逻辑改动 **BREAKING**
- **报告格式改为 Excel (.xlsx)**，自动保存到本地对应的归档文件夹
- **对账详情页增加「下载报告」按钮**
- **报告分为 3 种**：
  1. **自动入账凭证报告** — 完全匹配成功的数据，Excel 字段：
     | 序号 | 交易日期 | 银行流水信息(资金流) | | 关联单据信息(业务流) | | 核销结果(AI产出) | | |
     | | | 对方户名/摘要 | 到账金额 | 客户名称/单据号 | 应收金额 | 核销金额 | 差额 |
  2. **可解释性报告** — 匹配成功但存在可解释性差异的数据，Excel 字段：
     | 关联序号 | AI匹配逻辑(Reasoning Chain) | 证据链(Evidence) | 置信度 | 状态 |
  3. **异常情况处理报告** — 比对中存在异常的数据，Excel 字段：
     | 风险等级 | 异常类型 | 银行流水详情 | AI诊断分析 | AI建议操作 |
- **报告命名规则**：`{日期}-{当天比对次序}{报告名称}`，例如 `20260213-3自动入账凭证报告.xlsx`

## Capabilities

### New Capabilities
- `folder-auto-structure`: 用户选择一个根文件夹后，系统自动生成标准化子文件夹结构（00归档、01发票、02银行流水），并管理归档子文件夹的自动创建和命名
- `excel-report-generation`: 基于 3 种标准化 Excel 模板（自动入账凭证、可解释性、异常情况处理）生成对账报告，支持自动保存到归档文件夹和前端下载

### Modified Capabilities
- （无现有 specs 目录，所有变更以 New Capabilities 或 UI 改动形式呈现）

## Impact

### 受影响的代码文件
- `src/pages/Home/index.tsx` — UI 大幅简化，移除统计模块和映射导入
- `src/pages/Home/Home.module.scss` — 相应样式调整
- `src/pages/Reconciliation/index.tsx` — 移除状态标签和创建批次功能
- `src/pages/Reconciliation/Reconciliation.module.scss` — 相应样式调整
- `src/pages/ReconciliationDetail/index.tsx` — 匹配统计数据重构、移除批次信息
- `src/components/FolderConfigModal/index.tsx` — **重构**：改为只选择一个根文件夹，自动生成子文件夹
- `src/pages/Settings/index.tsx` — 新增文件夹重置功能
- `electron/main.ts` 或相关主进程文件 — 新增文件夹自动创建、归档转存、Excel 报告生成逻辑

### 受影响的 API / IPC 接口
- `electron.config.set/getAll` — 配置模型变更（从 3 个文件夹路径改为 1 个根路径）
- `electron.reconciliation.generateReport` — 报告生成逻辑重构为 Excel 格式
- `electron.file.scanFolder` — 扫描逻辑适配新文件夹结构
- 新增 IPC：`electron.folder.initStructure` — 初始化文件夹结构
- 新增 IPC：`electron.folder.archiveFiles` — 对账完成后归档文件

### 依赖
- 需要引入 Excel 生成库（如 `xlsx` 或 `exceljs`），用于生成 `.xlsx` 格式报告
